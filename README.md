# 工时日志一键导出 - 油猴脚本 (Tampermonkey) 使用教程

本脚本旨在帮助用户从工时系统（http://172.20.10.80/hr/work/workLogmy?activeTab=dispatch）实现“所见即所得”的批量导出至 CSV 文件。

## 🎬 功能亮点
✅ **完全纯前端处理**：无需知道后端接口，脚本模拟翻页，获取您在屏幕上切实看到的所有数据。
✅ **直观页码控制**：自由设定抓取从哪一页到哪一页，全自动点击翻页。
✅ **智能命名**：导出的 CSV 文件会自动提取出数据里**最老的日期与最新的日期**来充当文件名，管理一目了然！

---

## 🛠️ 安装步骤

### 第一步：安装浏览器扩展 Tampermonkey（篡改猴）
如果您还没有安装该插件，请前往对应浏览器的应用商店下载安装：
- **Edge** 浏览器：去扩展商城搜索 `Tampermonkey` 添加。
- **Chrome** 浏览器：去 Chrome Web Store 搜索 `Tampermonkey` 添加。

### 第二步：添加脚本
1. 点击浏览器右上角**Tampermonkey 黑色的图标**，选择由下方弹框里的 **“添加新脚本” (Create a new script...)**。
2. 此时会打开一个代码编辑器页面。**删除框里原有的所有文字代码**。
3. 把下方的 `worklog_export.user.js` 的完整代码**复制并粘贴**进去。
4. 按 `Ctrl + S` （或点击左上角 文件 -> 保存）。
5. 回到“已安装脚本”列表中，确保“工时日志一键导出”右侧绿色的开关已开启。

### 第三步（关键）：配置插件权限
出于浏览器安全机制，我们需要确保 Tampermonkey 具有充分的权限：
1. **浏览器扩展权限设置**：
   - 在 Chrome 浏览器中，右键点击右上角的 Tampermonkey 图标，选择 **“管理扩展程序”**。
   - 确保 **“允许访问文件网址”**（Allow access to file URLs）这一项已**开启**。（*如果您通过本地文件拖拽安装该脚本，此项必开*）
2. **下载权限（@grant GM_download）**：
   - 因为脚本执行完抓取后需要帮您把内容直接下载为 `.csv` 文件，这需要用到 `GM_download` 的越权下载 API。
   - 一般情况下，第一次弹窗导出 CSV 文件时，Tampermonkey 可能会在网页顶部或右上角弹出一个**“运行权限请求”提示框**。
   - 请务必在弹窗中选择 **“总是允许”（Always allow）**。如果错过了这个弹窗导致无法下载，去“已安装脚本 -> 点击此脚本名称 -> 设置 -> XHR 安全”中将权限放行。

## 🚀 怎么使用

1. 打开平时填写工时日志的页面并登录。由于系统结构复杂，需确保您已点进了上方的 **工时记录(dispatch)** 标签。
2. 等待网页数据完全加载完毕（约2秒），此时在网页原本的区域，就会多出一个带有浅灰色底色的组件：
   > 导出页码： `[ 1 ]` 至 `[ 10 ]` 页 [ 一键导出 ]

3. **配置参数**
   - 框内数字默认为 `1-10页`，请将其修改为您真实的翻页区间（例如你想导出第 `4` 到 `20` 页，就填入。**切记结束页不能小于起始页**）。

4. **开始抓取**
   - 点击绿色的【一键导出】按钮。
   - 会弹出确认提示框核对一次页码，点击“确认”。
   - 接下来完全放开鼠标！脚本会自动帮您疯狂点击**【下一页】**甚至自动**跳转到起始页**，中途不要干扰。
   - 数据抓取完毕后，会弹窗提示总条数，紧接着浏览器自动弹出 `工作日志_XXXX-XX-XX_至_XXXX-XX-XX.csv` 的下载项。

---

## ⚠️ 避坑指南

- 如果刚打开网页没刷出按钮，可以按 `F5` 刷新一下，因为单页面路由切换需要延时检测。
- 在脚本努力翻页为您搜集表单的期间（按钮显示：`正在抓取第 X 页...`），**请不要离开当前标签页或者手动去乱点它的页码！不要切去聊微信！**（有些浏览器机制下失焦可能导致后台停止运行或网速中断，从而卡住）。
- 如果你填了 1 到 500 页，但系统里统共才只有 30 页。别担心，脚本会在翻不过去下一页时自动识别并安全停止打印出前面的这部分数据。
